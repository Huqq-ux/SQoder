# Python Web 应用部署 SOP

## 概述
本文档描述了将 Python Web 应用部署到服务器的标准操作流程。

## 适用场景
- 新应用首次部署
- 应用版本更新部署
- 环境迁移部署

## 前置条件
- 服务器已安装 Python 3.9+
- 拥有项目代码仓库访问权限
- 已获取部署环境配置信息

## 操作步骤

### 步骤1：环境检查
检查 Python 版本和系统依赖是否满足要求：
- 执行 `python --version` 确认 Python 版本 >= 3.9
- 执行 `pip --version` 确认 pip 可用
- 检查系统是否安装了必要的编译工具（gcc、make 等）

### 步骤2：代码准备
获取项目代码并安装依赖：
- 克隆或拉取最新代码到部署目录
- 创建 Python 虚拟环境：`python -m venv venv`
- 激活虚拟环境并安装依赖：`pip install -r requirements.txt`

### 步骤3：配置管理
创建和配置环境变量文件：
- 根据部署环境创建对应的 `.env` 文件
- 配置数据库连接字符串
- 配置 API 密钥和第三方服务凭证
- 设置 FLASK_ENV 或 APP_ENV 环境变量

### 步骤4：数据库迁移
执行数据库迁移脚本：
- Django 项目：`python manage.py migrate`
- FastAPI 项目：`alembic upgrade head`
- 确认迁移执行成功，无错误输出

### 步骤5：服务启动
启动应用服务：
- 开发环境：`python app.py` 或 `flask run`
- 生产环境：使用 `gunicorn` 或 `uvicorn` 启动
- 确认服务在指定端口正常监听

### 步骤6：健康检查
验证服务是否正常运行：
- 访问 `/health` 或 `/api/health` 端点
- 确认返回 HTTP 200 状态码
- 检查返回的健康状态信息

## 注意事项
- 部署前务必备份当前版本
- 生产环境建议使用蓝绿部署或滚动更新
- 关键操作需要双人确认

## 回滚方案
如果部署失败：
1. 停止新版本服务
2. 恢复备份的代码和配置
3. 重启旧版本服务
4. 验证旧版本正常运行
